AIDE Steps
1
Open AIDE on Android
Install from Play Store if needed.
2
New Project → Android App
Tap the menu → New Project → Android App
3
Package Name
When prompted enter exactly:
com.acmeproducts.sayit
4
Replace MainActivity.java
Find the default MainActivity.java in the AIDE project tree. Delete it. Upload the MainActivity.java you downloaded from step 2. The URL is already correct — no editing needed.
5
Replace AndroidManifest.xml
Same process — delete default, upload your generated AndroidManifest.xml.
6
Add AppCompat Dependency
Open build.gradle. In the dependencies block add:
implementation 'androidx.appcompat:appcompat:1.6.1'
7
Build & Install
Tap the Run button in AIDE. It compiles and prompts to install.
Package name is permanent after install. Use exactly what's shown above. For a new repo, install a new APK with a different package name.
Multiple Versions / Subdirs
One APK serves all versions in one repo. Switch the active subdir inside the app (home.html gear icon → change subdir). For a different repo, run this workbook with new config and install a separate APK.

Adding a New Version Later
Run this workbook again with a different subdir (e.g. say-it-v2). Generate. Upload new files to the new subdir. Switch the active subdir in the app settings. Same APK, no reinstall.
Verification Checklist
Run through these after deploying.
Common Issues
404 on GH Pages URL — Check subdir path. Pages must be enabled. Wait 3 min after first deploy.
Voice not working — Grant Microphone to the APK: App Settings → Permissions → Microphone.
PAT rejected — Regenerate with Contents read+write. Check it hasn't expired.
AI platform won't open — Must be logged in inside the APK browser.
Blank screen in APK — The Java already has the correct URL from your config. Verify GH Pages is live first.
Config not showing in app — APK injects config from SharedPreferences into localStorage on every page load. Verify via home.html gear icon.

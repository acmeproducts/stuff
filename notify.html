Purpose

This demo answers a precise question: what notification and attention mechanisms are realistically available to a hosted HTML app when the page is focused, when another tab is active, and when the browser is open but another app is in the foreground.

The page demonstrates both the visible-in-page methods and the stronger system-level methods. It also distinguishes the two notification paths that matter in practice on mobile Chrome:

Page-side constructorOften blocked on mobile
Service worker notificationRequired for reliable mobile path
Blob worker timingUseful while page stays alive

Interpretation

For a chat app, the reliable stack is layered:

Focused pageBanner + sound + unread marker
Another tab activeTitle + favicon + optional system notification
Browser behind another appService-worker notification is the strongest path
Return to pageUnread indicator remains the durable fallback

Setup

Expected: permission prompt appears once; service worker registration succeeds on HTTPS or GitHub Pages; constructor capability is classified explicitly instead of logged as a generic failure.

Focused-page cues

Expected: while this tab is focused you should see the in-page banner, favicon badge, title flash, unread count increase, and usually an audio cue. No background delivery is being proven here.

Notification paths

Expected: the page-constructor path may work on some desktop browsers but is expected to be unavailable on many mobile Chrome runs; the service-worker path is the proper Chrome mobile path and should be the one you trust operationally.

Delayed tests

Expected: the blob worker can schedule delayed events while the page remains alive. If the page is hidden, the in-page cues will still update internally; whether you notice them depends on browser state. This does not prove survival through aggressive page suspension.

Reset

Expected: unread count, title, favicon badge, banner, and app badge attempt reset to baseline.

Event log

This log is intended to distinguish expected platform behavior from actual regressions. Constructor-blocked notification paths are labeled as an expected limitation when detected.
